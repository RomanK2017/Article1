Road markings recognition and classification

 

I don’t often publish anything on LinkedIn but this project I really enjoyed. It was an attempt to create from scratch a system which could detect and classify road markings. At the moment this system detects 16 classes of road markings, the data and model are constantly updated.

 It is very common to use CNN (convolutional neural network) for filtering and classification. I could use resized contour images and CNN but there is a problem. Image resizing will cause the loss of important information about original contour size, sometimes the contour size was the only way to define the difference between classes. It means the input parameters of a classification model must be updated.

The alternative approach is to use a mathematical way of shape description of contour such as Hu and Zernike moments or Fourier descriptors.

I decided to combine both ways and represent the input parameter as a vector: [7 Hu moments (shape descriptors),A(contour area), hx, hy(length and height of contour), k,l (linear orientation of contour), 256 pixel brightness components (resized 16x16 grey contour image) ]-> 7+5+256 = 268 vector components.

Picture 4 – Contour image (A), resized 16x16 contour image (B), array of pixel brightness of 16x16 image (C)

 In demonstration mode the system colours each detected contour; then labels the contour’s number and name of the class. There is ‘Rubbish’ class which helps filter out incorrect contours caused by cars, shadows, bright spots, etc. 

Picture 5 – All detected contours

Picture 6 – Image without ‘Rubbish’ contours

 The best performance was achieved with multilayer perceptron neural network (acc. on validation dataset >85%) The dataset contains a few thousand examples, each class at least 500.
